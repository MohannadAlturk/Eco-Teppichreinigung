PROJEKTBESCHREIBUNG – ECO-TEPPICHREINIGUNG PLATTFORM


MEINE VORGEHENSWEISE
═══════════════════════════════════════════════════════════════

1. ANALYSE & PLANUNG
   - Ich teile die Anwendung in einzelne Screens/Seiten auf
   - Für jeden Screen schaue ich mir an, welche Features reinmüssen
   - Ich plane die Struktur und überlege, wie alles zusammenspielt


2. TECHNOLOGIE-ENTSCHEIDUNG

   Für das Frontend habe ich mich entschieden für:
   - React mit TypeScript (moderne, typsichere Entwicklung)
   - Next.js (professionelles Framework)
   - Tailwind CSS (schnelles, modernes Design)
   - Zustand (einfaches State Management)

   Warum diese Technologien?
   - Professioneller Standard
   - Schnelle Entwicklung
   - Leicht erweiterbar für später
   - Gute Performance


3. FRONTEND ENTWICKLUNG (ERSTE PHASE)

   Ich starte mit dem Frontend und entwickle:

   • Landing Page
     - Hero-Bereich mit Call-to-Action
     - Vorteile zeigen (umweltfreundlich, kostenloser Versand, etc.)
     - Prozess in 3 Schritten erklären

   • Login & Registrierung
     - Anmeldeformular
     - Registrierungsformular
     - Passwort vergessen Funktion

   • Teppich-Konfigurator
     - Schritt 1: Teppichart auswählen
     - Schritt 2: Größe eingeben
     - Schritt 3: Dicke angeben
     - Schritt 4: Zustand beschreiben
     - Schritt 5: Fotos hochladen
     - Schritt 6: Preis berechnen

   • Warenkorb & Checkout
     - Produkte im Warenkorb anzeigen
     - Adresseingabe
     - Versandoption
     - Zahlungssimulation

   • Dashboard
     - Bestellübersicht
     - Bestelldetails mit Status-Timeline
     - Profilverwaltung
     - Einstellungen


4. PROTOTYP-BESONDERHEITEN

   Der Prototyp arbeitet zunächst OHNE Backend:
   - Alle Daten sind Mock-Daten (Beispieldaten)
   - Keine echte Datenbank
   - Keine echte Zahlung
   - Aber: Alles ist klickbar und funktioniert

   Vorteil:
   - Schnelle Demo möglich
   - Sie können alles testen und ausprobieren
   - Später kann echtes Backend angebunden werden


5. BACKEND (ZWEITE PHASE)

   Nach dem Frontend komme ich zum Backend:
   - Datenbank aufsetzen
   - API entwickeln
   - Authentifizierung einbauen
   - Zahlungsanbieter anbinden
   - Alles mit dem Frontend verbinden


6. DESIGN-PRINZIPIEN

   Ich achte auf:
   - Modernes, minimalistisches Design
   - Klare Struktur
   - Einfache Bedienung
   - Professioneller Look
   - Mobile Optimierung (funktioniert auf allen Geräten)


7. PROJEKT-STRUKTUR

   Ich organisiere den Code sauber:
   - /app = Alle Seiten
   - /components = Wiederverwendbare Bausteine
   - /data = Beispieldaten
   - /store = Datenverwaltung
   - /types = Typdefinitionen
   - /utils = Hilfsfunktionen
